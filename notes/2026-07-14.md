## What surprised us

- **`--eval.use_async_envs` (the default, `True`) crashes on this machine.**
  `AsyncVectorEnv._check_spaces()` hit a `BrokenPipeError` during the
  multiprocessing handshake — almost certainly a pygame/pymunk object in
  the PushT env that doesn't survive being pickled across a `spawn`-started
  subprocess on macOS. Fix: `--eval.use_async_envs=false` (`SyncVectorEnv`,
  single process, still batches the policy's forward pass across envs).
  Worth remembering for any future eval run on this machine — the default
  doesn't work here.
- **`lerobot-eval` hardcodes a video-rendering cap.**
  `max_episodes_rendered = 0 if cfg.eval.recording else 10`
  (`lerobot_eval.py:772`) — regardless of `n_episodes`, only the first 10
  episodes ever get an MP4. All 50 episodes' reward/success land in
  `eval_info.json`, but `video_paths` only has 10 entries. This is
  intentional upstream behavior (not a bug), but it exposed a real bug in
  our own portal code: `run_detail()` was iterating over `video_paths` to
  build the episode list, so episodes 10–49 were silently dropped from the
  UI entirely. Fixed by iterating over `successes` (the full-length array)
  instead, with `video_url`/`length_steps` set to `null` for episodes
  without a rendered video. **The lesson: when two arrays in the same
  payload are supposed to have equal length and one turns out shorter, the
  bug is in whichever code assumed they'd always match — verify that
  assumption instead of trusting it.**
- **Wall time was ~11.2 hours** (`eval_s = 40400.9`), against a rough
  estimate of ~1 hour for 50 episodes at the batched rate. This spans the
  reported per-step timings (`2.5s/it` to `3.7s/it` in the tail of the log,
  vs. ~0.64s/step measured in the n=5 run) — inconsistent with steady
  compute, and consistent with the laptop sleeping/idling partway through
  an unattended overnight run. **Not treating this as a real inference-speed
  number** — see open questions.

  episodes have rendered video (see the `max_episodes_rendered` note above).
